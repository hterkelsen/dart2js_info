Properties

classes → List<ClassInfo>
read / write

Information about each class (in any library).

constants → List<ConstantInfo>
read / write

Information about constants anywhere in the program.

deferredFiles → Map<String,Map<String>>
read / write

Details about all deferred imports and what files would be loaded when the import is resolved.

dependencies → Map<Info,List<Info>>
read / write

A new representation of dependencies form one info to another. An entry in this map indicates that an Info depends on another (e.g. a function invokes another). Please note that the data in this field might not be accurate yet (this is work in progress).

fields → List<FieldInfo>
read / write

Information about fields (in any class).

functions → List<FunctionInfo>
read / write

Information about each function (includes methods and getters in any library)

libraries → List<LibraryInfo>
read / write

Information about each library processed by the compiler.

minorVersion → int
read-only

Minor version indicating non-breaking changes in the format. A change in this version number means that the json parsing in this library from a previous will continue to work after the change. This is typically increased when adding new entries to the file format.

outputUnits → List<OutputUnitInfo>
read / write

Information about output units (should be just one entry if not using deferred loading).

program → ProgramInfo
read / write

Summary information about the program.

typedefs → List<TypedefInfo>
read / write

Information about type defs in the program.

version → int
read-only

Major version indicating breaking changes in the format. A new version means that an old deserialization algorithm will not work with the new format.
